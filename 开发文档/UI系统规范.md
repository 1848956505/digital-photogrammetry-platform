# UI系统规范

本文档定义本项目主界面的 UI 职责边界、中央展示区架构、对比模式、3D 视图、模块接入方式以及界面中文化规范。后续所有模块与页面开发均应遵循本规范。

## 1. 总体目标

主界面应从“按钮 + 结果文本”的形式，升级为“工作台式可视化界面”。

核心目标如下：
- 中央区域负责空间结果可视化
- 右侧区域负责参数控制、结果摘要与数值指标
- 左侧区域负责数据对象与图层管理
- 底部区域负责日志、状态、进度与坐标信息
- 所有界面文本默认使用中文显示
- 需要直观理解的位置、形态、覆盖范围类结果，必须进入中央展示区

## 2. 界面语言规范

### 2.1 默认语言

项目界面默认语言为中文简体。

### 2.2 中文化要求

以下内容必须使用中文显示：
- 按钮文字
- 分组标题
- 菜单项
- 标签文字
- 提示信息
- 状态信息
- 错误提示
- 成功提示
- 对比模式标题
- 3D 视图标题
- 中央展示区空状态提示

### 2.3 英文使用规则

英文仅在以下情况允许出现：
- 算法或方法名需要保留原名，例如 `SIFT`、`ORB`、`RANSAC`
- 文件格式或标准名称需要保留原名，例如 `TIFF`、`GeoTIFF`
- 技术约定俗成的专业名词需要保留原名

### 2.4 文案原则

- 用户看到的主要界面文本应尽量中文化
- 专业术语允许中文 + 英文并列，例如“特征提取（SIFT）”
- 不要在核心按钮、模式切换、结果区使用纯英文
- 后续模块新增控件时，应优先使用中文命名

## 3. 中央展示区职责

中央展示区是系统的核心结果承载区，只负责空间结果可视化，不承担复杂参数输入。

### 3.1 支持的三种模式

中央展示区必须支持以下三种模式：
- `2D` 视图
- `对比` 视图
- `3D` 视图

### 3.2 2D 视图职责

2D 视图用于显示二维图像类结果，包括：
- 原始图像
- 处理后图像
- 关键点
- 匹配线
- 掩膜 `mask`
- 分割结果
- DOM 结果
- 图像对比结果

2D 视图应支持：
- 单图显示
- 图层叠加
- 关键点叠加
- 掩膜半透明叠加
- 滚轮缩放
- 鼠标拖动平移
- 图层开关
- 必要时的同步浏览

### 3.3 对比视图职责

对比视图不是独立技术栈，而是 2D 视图的分屏模式。

对比视图用于显示：
- 原图 vs 处理后
- SIFT 匹配 vs ORB 模拟匹配
- 不同重采样方法对比
- 不同滤波结果对比
- 后续模块的多结果对比

对比视图必须满足：
- 左右两侧稳定显示两张图
- 左右视图都支持缩放和平移
- 支持联动浏览
- 支持分别叠加各自的关键点、掩膜、匹配线
- 不得使用多个临时弹窗替代对比模式

### 3.4 3D 视图职责

3D 视图用于显示三维空间结果，包括：
- 点云
- DSM / DEM
- 三维重建结果
- 相机位姿

3D 视图应至少支持：
- 显示点云
- 显示 DSM / DEM 表面
- 旋转
- 缩放
- 平移

## 4. 右侧面板职责

右侧面板是控制与摘要区域，不是主要结果展示区。

### 4.1 右侧面板适合放置的内容

- 算法选择
- 参数输入
- 开始按钮
- 结果摘要
- 数值指标
- 简短说明
- 运行时状态

### 4.2 右侧面板不应承担的内容

- 特征点图
- 匹配连线图
- 分割结果图
- 重采样多结果对比图
- 3D 结果图
- 需要依赖空间位置理解的结果图

### 4.3 右侧摘要内容建议

- 特征点数量
- 匹配点数量
- 内点数量
- 内点率
- 重投影误差
- 处理耗时
- 输出文件路径
- 当前运行状态

## 5. 左侧图层与数据树职责

左侧区域负责管理数据对象和图层顺序，不承担算法说明。

### 5.1 左侧区域适合展示的内容

- 原始影像
- 处理结果
- 掩膜
- 点云
- 矢量
- DSM / DEM
- DOM
- 结果图层开关
- 图层顺序控制

### 5.2 左侧区域职责原则

- 左侧用于对象管理
- 左侧用于显示和隐藏控制
- 左侧用于图层顺序控制
- 左侧不显示大段算法说明
- 左侧不承载复杂参数表单

## 6. 底部日志与状态栏职责

底部区域负责运行过程信息和交互状态。

### 6.1 底部区域适合展示的内容

- 运行日志
- 进度信息
- 状态提示
- 当前鼠标坐标
- 当前像素值
- 当前模式
- 成功或失败提示

### 6.2 底部区域不应承担的内容

- 最终结果图
- 复杂参数输入
- 大段说明文本
- 完整分析报告

## 7. 中央展示区实现要求

### 7.1 2D 视图实现要求

2D 视图必须支持以下叠加图层：
- 关键点图层
- 匹配线图层
- 掩膜图层
- 结果图层
- 显示 / 隐藏控制

### 7.2 对比视图实现要求

对比视图必须基于 2D 分屏实现，不能单独再造一套显示框架。

### 7.3 3D 视图实现要求

3D 视图必须是最小可用版本，不能长期保持占位符状态。

## 8. 模块接入规范

### 8.1 必须进入中央视图的结果

以下结果必须优先进入中央视图：
- 关键点
- 匹配线
- 掩膜
- 图像增强前后对比
- 重采样对比
- 点云
- DSM / DEM
- 三维重建结果
- DOM 图像

### 8.2 只适合放在右侧的内容

以下内容只适合放在右侧：
- 参数输入
- 算法选择
- 数值摘要
- 简短解释
- 运行状态

### 8.3 必须进入 3D 视图的内容

以下内容必须进入 3D 视图：
- 点云
- DSM / DEM
- 三维重建结果
- 相机位姿

### 8.4 必须复用对比模式的需求

以下需求必须复用对比模式：
- 原图 vs 处理后
- SIFT vs ORB 模拟匹配
- 不同插值方法对比
- 不同滤波方法对比
- 后续模块的多结果空间对比

## 9. 模块一接入要求

### 9.1 特征提取

特征提取结果必须在中央 2D 视图中显示关键点叠加图，同时右侧显示数值摘要。

### 9.2 特征匹配

特征匹配结果必须进入对比模式，左、右两侧显示匹配图，并区分内点与外点。

如果当前并未接入真实深度学习匹配模型，界面文案必须明确写成：
- `ORB（模拟对比）`
- 或其他清晰的模拟表述

不得误导用户认为已经接入 SuperGlue、LoFTR 等真实模型。

### 9.3 重采样

重采样结果必须支持多方法对比，至少应能看到原图与不同插值结果之间的差异。

### 9.4 灰度变换与滤波

灰度变换与滤波类功能应支持前后对比显示，不能只在右侧显示“处理完成”。

## 10. 现阶段实现原则

### 10.1 当前实现约束

当前版本必须优先保证：
- 中央区域可切换 `2D / 对比 / 3D`
- 2D 视图可叠加关键点和掩膜
- 对比视图可稳定显示两张图
- 3D 视图可稳定显示点云或表面
- 界面默认中文显示

### 10.2 未来扩展预留

未来可继续增强：
- 2D 图层更多交互
- 对比视图历史记录
- 3D 相机位姿联动
- 更复杂的点云渲染
- 更细粒度的图层管理

## 11. 验收标准

本规范落地后，应满足以下条件：
- 界面文本默认中文显示
- 中央区域具备可用的 `2D / 对比 / 3D` 三类展示能力
- 模块一的关键结果不再只显示在右侧文字区
- 后续模块可直接复用这套展示框架
- 各 UI 区域职责清晰，不再混用
