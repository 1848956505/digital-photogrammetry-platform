### 一、 整体版面布局规划 (Wireframe)

整个主窗口（QMainWindow）分为 **5 个核心区域**，除了中央视图外，周边面板均可拖拽、悬浮或隐藏。

```
+-----------------------------------------------------------------------+
|  顶部区域：动态菜单栏 (Menu Bar)  &  工具集 (Ribbon/Toolbar)               |
+---------------+---------------------------------------+---------------+
|               |                                       |               |
|  左侧区域：     |  中央区域：                             |  右侧区域：     |
|  数据与图层     |  多视图工作区 (Multi-Tab Workspace)      |  动态工具参数    |
|  管理器         |  [ 2D 视图 ]  [ 3D 视图 ] [ 对比视图 ]   |  与属性面板     |
|  (Data &      |                                       |  (Properties) |
|   Layers)     |      显示影像、矢量图、点云、掩膜等          |               |
|               |                                       |               |
+---------------+---------------------------------------+---------------+
|  底部区域：控制台输出 (Console/Log) & 状态栏 (Status Bar - 坐标、进度)    |
+-----------------------------------------------------------------------+
```

### 二、 各区域详细设计与“插件化”响应

#### 1. 顶部区域 (Top)：动态菜单与工具栏

- **设计理念**：采用 Ribbon 风格（类似 Office）或标准菜单栏。这是**插件系统的核心挂载点**。
- **动态响应**：系统启动时，只有基础的“文件(File)”、“视图(View)”。如果扫描到“模块一”插件，工具栏自动生成一个名为【基础处理】的页签，内含“灰度变换”、“SIFT提取”等按钮。如果删除了“模块六”，【AI 解译】的页签和按钮就完全不显示。

#### 2. 左侧区域 (Left)：图层与数据树 (Layer Manager)

- **设计理念**：一个树状列表（Tree View），用于管理导入和生成的各类数据。
- **节点分类**：📂 **影像 (Images)**：原始 TIFF、拼接后的 DOM。📂 **点云与高程 (Point Clouds & DEM)**：空三点云、滤波后点云、DSM/DEM。📂 **矢量 (Vectors)**：模块五 (DLG) 绘制的 Shapefile，或模块六生成的解译边界。
- **交互**：每个数据前有一个“眼睛”图标（👁️），点击即可在中央工作区显示或隐藏。支持拖拽调整图层上下叠加顺序。

#### 3. 中央区域 (Center)：多视图工作区

这是最核心的显示区域，采用**多选项卡（Tab）**设计，支持随时切分屏幕（Split View）。

- **【2D 地图视图】**：用于显示影像、DOM、DLG。支持鼠标滚轮无极缩放、中键平移。**模块六（UNet解译）交互**：AI 输出的语义分割结果（如建筑、水体）以半透明遮罩（Mask）形式叠加在 2D 原图上，提供一个“透明度滑块”。
- **【3D 场景视图】**：用于显示空三解算结果（相机位姿金字塔）、密集点云、DSM/DEM 三维地形。支持三维旋转（Orbit）、平移、缩放。**模块三（AI点云滤波）交互**：将 PointNet 识别的地面点染成棕色，非地面点染成绿色，直观展示 3D 滤波效果。
- **【双屏对比视图】（专为模块一定制）**：左右切分屏幕。左边图 A，右边图 B。**模块一实战演示**：点击“SIFT与AI匹配对比”后，屏幕出现同名点连线（Tie Lines）。左侧显示传统 SIFT 连线，右侧显示深度学习连线，方便直观汇报效果。

#### 4. 右侧区域 (Right)：动态参数/工具面板

- **设计理念**：为了避免满屏幕弹窗（Pop-up Dialog），所有的算法参数设置都在右侧面板完成。
- **动态响应**：当在顶部点击了【模块二：间接平差法】时，右侧面板自动切换为平差参数UI（如迭代次数、收敛阈值设定），底部有一个大大的“运行(Run)”按钮。当点击【模块四：影像色彩调整】时，右侧面板变成匀色算法选择（直方图匹配/Wallis）和参考影像选择框。当点击【模块七：MipMap三维重建】时，右侧面板显示该模块的 Markdown 说明文档（内嵌网页视图），以及一个“调用外部组件”按钮。

#### 5. 底部区域 (Bottom)：日志与状态栏

- **控制台输出 (Log)**：滚动显示系统信息。例如：“正在加载UNet模型...”、“平差完成，中误差为0.5像素”。对于深度学习任务，这里极具极客感。
- **状态栏 (Status)**：左侧显示鼠标指针在 2D/3D 视图中的实时坐标 `(X,Y,Z)(X,Y,Z)` 和像素灰度值。右侧放置一个全局进度条（Progress Bar），当后台运行耗时算法（如密集匹配、PointNet推理）时，进度条滚动。

### 三、 模块化 UI 落地技术选型 (基于 PySide6 / PyQt5)

如果您使用 Python，强烈建议使用 **PySide6**，UI 开发极其方便，且可以完全实现上述效果：

1. **主框架**：使用 QMainWindow。
2. **停靠面板**：左侧和右侧面板使用 QDockWidget，用户可以将面板拖出来放在副显示器上，非常适合专业工具。
3. **2D 渲染引擎**：使用 QGraphicsView 和 QGraphicsScene。对于超大遥感 TIFF 影像，结合 GDAL 进行分块金字塔读取（类似瓦片地图），避免内存撑爆。
4. **3D 渲染引擎**：使用 pyqtgraph 的 3D 模块，或者集成开源的 **Open3D** 视图，或者使用 QOpenGLWidget。
5. **图层树**：使用 QTreeView 结合自定义的 QAbstractItemModel。
6. **UI 风格 (Theme)**：引入 **QDarkStyleSheet** 或 **Qt Material** 主题包，一键让软件变成高级的深色模式（Dark Mode），这在测绘和深度学习软件中非常流行，显得专业且保护视力。
