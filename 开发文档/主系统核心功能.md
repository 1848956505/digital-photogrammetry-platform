# 主系统核心功能架构完整方案

## 架构总览视图

主系统（Core）由以下 8 个相互独立的子系统构成：

1. **Workspace (数据总管)**：管理全生命周期的数据。
2. **Project Manager (序列化)**：项目的保存与读取。
3. **2D Render Engine (二维引擎)**：支持多图层、漫游、测量的视图。
4. **3D Render Engine (三维引擎)**：点云与高程的三维展示。
5. **Task Engine (异步多线程)**：防止 UI 卡死的后台计算池。
6. **Event Bus (事件通信)**：解耦模块间通信的消息中枢。
7. **IO Manager (文件输入输出)**：标准化各类测绘数据的导入。
8. **UI Shell (用户界面壳)**：主窗口联动、状态跟踪、日志与进度反馈。

## 核心子系统详细设计规范

### 1. 全局数据总管 (Workspace)

- **定位**：内存中的“中央数据库”。所有插件不直接读写本地文件，而是向 Workspace 要数据。
- **设计规范**：维护字典结构：self.data = {"images": {}, "point_clouds": {}, "vectors": {}, "masks": {}}。支持数据的增删改查（CRUD）接口：add_data(type, id, path/array)。**缓存管理**：对于几 GB 的 TIFF 影像或千万级点云，Workspace 只存“文件路径”或“内存映射（Memmap）”，避免内存溢出（OOM）。

### 2. 项目持久化中心 (Project Manager)

- **定位**：让用户的工作可以保存，明天继续干。
- **设计规范**：将 Workspace 中的数据路径、当前插件的参数设置，打包序列化为 .json 或 .sqlite 格式的工程文件（如 project.gai）。实现 **“打开工程”、“保存工程”、“另存为”** 功能。工程加载时，自动重构左侧的 QTreeWidget 目录树。

### 3. 高级 2D 渲染引擎 (2D Render Engine)

- **定位**：支撑模块一（影像处理）、模块四（DOM）、模块五（DLG）、模块六（UNet）的核心画布。
- **设计规范**（基于 QGraphicsView）：**无限漫游**：必须重写 wheelEvent (滚轮以鼠标指针为中心缩放) 和 mouseMoveEvent (中键平移)。**多图层堆叠 (Layer Stack)**：支持 BaseLayer（底图）、VectorLayer（DLG矢量层）、MaskLayer（半透明的UNet解译层）。**交互绘图机制**：支持通过鼠标在图层上绘制点、线、多边形（为模块五 DLG 采集做准备）。**大图瓦片加载 (可选/进阶)**：如果是超大遥感图，支持分块加载，不让 UI 卡顿。

### 4. 基础 3D 渲染引擎 (3D Render Engine)

- **定位**：支撑模块二（空三点云）、模块三（DSM/DEM滤波）、模块七（三维重建）的画布。
- **设计规范**：封装一个基于 Open3D.visualization.O3DVisualizer 或 pyqtgraph.opengl 的 3D 控件。支持点云文件（.las, .ply, .pcd）的加载与渲染。支持鼠标拖拽旋转（Orbit）、平移（Pan）、缩放（Zoom）。支持点云按高程（Z值）或分类颜色（如地面点、非地面点）渲染。

### 5. 异步多线程任务引擎 (Task Engine)

- **定位**：所有耗时算法的“后台健身房”。深度学习推理、空三平差绝不允许在主线程跑。
- **设计规范**（基于 QThreadPool 和 QRunnable / QThread）：定义标准任务类 WorkerTask。插件在执行耗时操作时，将函数打包成 WorkerTask 扔给 Task Engine。任务必须能对外发射 3 种信号：progress_updated (更新进度条)、log_emitted (输出控制台日志)、task_finished (任务完成，返回结果)。主界面底部状态栏全局监听进度信号，实时更新百分比进度条。

### 6. 全局事件总线 (Event Bus) - *新增核心功能*

- **定位**：让各模块之间“解耦”通信。例如模块一处理完图片，左侧树和右侧视图需要自动刷新，但它们之间不应该互相调用。
- **设计规范**：实现发布-订阅模式（Publish/Subscribe）。定义核心事件，如：DATA_ADDED，DATA_REMOVED，PLUGIN_FINISHED。当导入图片后，发送 DATA_ADDED 事件，左侧树形控件监听到该事件后自动增加一个节点；2D 视图监听到该事件后自动刷新画面。

### 7. 坐标与状态监测器 (Status & Coordinate Tracker)

- **定位**：测绘软件的灵魂，实时反馈空间信息。
- **设计规范**：视图引擎 (QGraphicsView / 3D Viewer) 捕获鼠标移动事件。实时进行坐标转换：屏幕坐标 -> 场景坐标 -> 图像像素坐标 `(col,row)(col,row)` -> 地理坐标 `(X,Y,Z)(X,Y,Z)`。将转换结果和当前像素的灰度/RGB值发送给 UI 层的主状态栏（StatusBar）实时显示。

### 8. 通用文件 IO 与解析器 (IO Manager)

- **定位**：统一管理不同后缀名数据的导入。
- **设计规范**：实现不同数据格式的 Factory 模式。图像类：使用 OpenCV/GDAL 解析 .tif, .png, .jpg。点云类：解析 .las, .ply。矢量类：解析 .shp, .geojson。

### 第三阶段：多线程任务引擎（防界面卡死）

**目标**：确保运行耗时算法（如模块一或模块六）时，主界面依然流畅。

**指令：**

**Sprint 3：异步任务管理与进度反馈**。遥感处理非常耗时，我们需要一个多线程引擎。

**任务要求：**

1. **创建 core/task_engine.py**：基于 QThreadPool 实现一个 TaskWorker。支持传入任何函数执行。拥有信号：started, finished, progress(int), error(str)。
2. **主窗口集成**：在主窗口底部状态栏（StatusBar）添加一个永久显示的 QProgressBar（默认隐藏）。
3. **模拟测试**：在“工具”菜单下创建一个“测试长耗时任务”按钮。点击后，通过 TaskWorker 在后台运行一个循环 5 秒的模拟计算，期间每秒更新一次进度条，且确保主界面仍能自由缩放影像、不卡顿。

请给出 task_engine.py 的实现以及主窗口如何调度任务的代码，同时更新readme文档。
